# 我给飞书加了个知识图谱

飞书知识库你用了多久了？

半年？一年？两年？

打开你的知识空间看一眼。几十篇文档安安静静躺在那里，像一个个抽屉——你往里面塞东西，但从不翻看抽屉之间有什么联系。

我也是。直到有一天我想找一篇半年前写的方案，翻了十分钟没找到，却意外发现：**三篇不同时间写的文档，居然在讨论同一个问题，而我完全不记得了。**

这让我意识到一件事——

**飞书给了你一个知识库，但没给你一张知识图谱。**

---

## 飞书缺了什么

用过 Obsidian 的人都知道，它有一个让人上瘾的功能：知识图谱。你能看到所有笔记之间的连接，一眼就知道哪些想法是相关的，哪些笔记是孤立的。

飞书没有这个。

飞书的文档之间，关系全靠你脑子记。你知道《Q1 规划》和《产品调研》有关系，但飞书不知道。你知道三篇会议纪要讨论的是同一件事，但飞书不知道。你甚至不知道自己有多少篇内容高度重复的文档——因为它们分散在不同的空间和文件夹里。

这不是飞书的 bug，而是一个**结构性的缺失**。

所以我决定自己做一个。

---

## 一句话就能跑

我做的这个工具叫 Knowledge Explorer（飞书知识探索器）。

它的使用方式特别简单——如果你用 Claude Code 或者 OpenClaw 这样的 AI Agent，装完之后直接说一句：

> "帮我分析一下知识库"

就完事了。不用配置 API key，不用写命令行参数，不用指定文档范围。AI 会自动扫描你的飞书知识空间，抓取文档内容，用 AI 理解每篇文档在说什么，然后从零开始建立文档之间的连接。

整个过程大概几分钟，取决于你的文档数量。

跑完之后，它会做两件事：
1. 在终端给你一份彩色摘要
2. **自动在飞书里创建一份完整的分析报告**

第一次看到那份报告的时候，说实话，我是有点惊讶的。

---

## 报告长什么样

它会把你的文档分成几类：

**枢纽文档** —— 被大量引用和关联的知识锚点。你可能知道《Q1 规划》很重要，但你不知道它和 8 篇文档有语义关联。

**孤岛文档** —— 和其他文档零关联。可能是写完就没人看过的会议纪要，也可能是一篇被遗忘的好文档。

**桥梁文档** —— 同时连接多个主题的文档。比如《用户调研》同时和产品组、市场组的文档相关，它是跨团队的知识纽带。

**过期文档** —— 被频繁引用但长期未更新。比如《竞品分析》已经三个月没动了，但还有 7 篇文档在引用它——该更新了。

然后它会把文档自动聚类成主题：

```
🔗 发现 5 个主题聚类
  ├ #阅读工坊 (8篇)
  ├ #社群运营 (7篇)
  ├ #暑期充电站 (7篇)
  ├ #活动策划 (5篇)
  └ #会议纪要 (4篇)
```

你从来没手动给这些文档打过标签，但 AI 读完内容后，自己把它们分好了组。

---

## 碰撞洞察：偶尔会有惊喜

报告的最后一部分，是我觉得最有意思的功能——**碰撞洞察**。

它会从不同主题群中，找到那些从未被关联过、但关键词有交集的文档对，然后问 AI：「如果把这两篇放在一起看，能产生什么新想法？」

比如：

> **《学员阅读痛点合集》×《核心思维课程整理》**
> → 围绕 Top 3 痛点重构 6 周课程大纲
>
> 文档A显示学员最大的卡点是"提取论证"——但课程把这个放在第 5 周，提前到第 1 周可能直接砍掉一半的早期流失。

> **《社群 SOP 执行记录》×《破冰活动指南》**
> → 把破冰环节嵌入社群入群 SOP 的 Day-1
>
> SOP 记录显示 40% 的新成员入群后沉默，而破冰活动的参与率达 90%。

说实话，这个功能**不是每次都能给出惊艳的建议**。有时候 AI 的碰撞会比较表面。但偶尔会蹦出一个让你"嗯？"的想法——那种你自己翻文档绝对不会想到的角度。

我觉得这就够了。知识管理不需要每次都有顿悟，偶尔的惊喜就是赚到。

## 怎么装

两种方式：

**方式一：AI Agent Skill（推荐）**

如果你用 Claude Code、OpenClaw 或其他 Coding Agent，把这段话发给它：

> 帮我安装飞书知识探索器（Knowledge Explorer）。按顺序执行：
> 1. 检查 Node.js >= 20，没有就提示我手动安装
> 2. `npm install -g @larksuite/cli`
> 3. `npm install -g lark-knowledge-explorer`
> 4. 运行 `lark-cli auth login` 授权飞书权限
> 5. 把 Skill 文件安装到 skills 目录

装完后说"帮我分析一下知识库"就行了。

**方式二：命令行直接跑**

```bash
npm install -g lark-knowledge-explorer
npx knowledge-explorer
```

需要自己配一个 AI API key（支持通义千问、DeepSeek、GPT 等）。

## 为什么做这个

最后说点背景。

这个工具是参加 lark-cli 创作者大赛做的。比赛要求基于飞书开放平台做开源项目，按 GitHub Stars 数计分。

但我做它的初始动机不是比赛——是我真的在用飞书写文档，真的遇到了"文档越来越多但越来越乱"的问题。比赛只是给了我一个把想法变成产品的理由。

项目完全开源，MIT 协议：

**GitHub：https://github.com/coni555/lark-knowledge-explorer**

如果你也用飞书，也被知识库的混乱困扰过，试一下。好用的话，帮忙点个 Star，对我这个参赛的大学生来说意义很大。
